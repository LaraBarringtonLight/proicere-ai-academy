proicere-academy / prompt-packs

Prompt Packs

Reusable, tested prompts from across the pods. Copy any prompt and adapt it to your context.

Controls
Risk register from meeting transcript
▼
Verified1–2 hrs savedJamie M. · Pod 1
You are a project controls specialist. I will paste a meeting transcript below. Your task: 1. Extract all mentioned risks, issues, and concerns 2. Categorise each as: Cost / Schedule / Scope / Resource / Quality / External 3. Assign an initial likelihood (Low / Medium / High) based on context 4. Suggest a brief mitigation action for each 5. Output as a structured table: ID | Risk | Category | Likelihood | Mitigation Only include items explicitly mentioned or strongly implied. Do not invent risks. [PASTE TRANSCRIPT BELOW]
Works best with transcripts over ~20 minutes. Always verify cost/budget figures manually.
Comms
Client update email from bullet notes
▼
Verified30–60 mins savedTom K. · Pod 1
Write a professional project update email using the notes below. Client name: [CLIENT NAME] Project: [PROJECT NAME] Tone: [formal / friendly-professional / technical] Send from: [YOUR NAME], [YOUR ROLE] Key points: [PASTE YOUR BULLET NOTES] Requirements: - Keep to under 250 words - Lead with progress, then issues, then next steps - Do not invent any information not in the notes - End with a clear call to action
Adjust the tone instruction carefully — "formal" and "friendly-professional" produce very different results.
Data
Power BI DAX measure generator
▼
Verified2+ hrs savedAlex M. · Pod 2
You are a Power BI expert. Write a DAX measure for the following: What I want it to calculate: [DESCRIBE IN PLAIN ENGLISH] My table name: [TABLE NAME] Relevant columns: [LIST COLUMNS] Any filters or conditions: [e.g. only rows where Status = "Active"] Please: 1. Write the full DAX measure with correct syntax 2. Explain in plain English what each part does 3. Flag any assumptions you've made 4. Suggest how to test it is working correctly
Always test in Power BI Desktop before deploying to a live report.
Comms
Executive summary from long report
▼
Verified1–2 hrs savedChloe P. · Pod 2
Summarise the following report into a concise executive summary. Audience: [senior leadership / client board / project sponsor] Max length: [1 page / 300 words / 5 bullet points] Structure: 1. Purpose of the report (1 sentence) 2. Key findings (3–5 bullet points) 3. Risks or issues flagged (if any) 4. Recommended actions or decisions required 5. Next steps / timeline Do not add information not present in the report. Use plain, non-jargon language. [PASTE REPORT TEXT OR UPLOAD PDF]
For very long reports (+50 pages), split into sections and summarise each, then summarise the summaries.
